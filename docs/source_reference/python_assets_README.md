# Asset Pipeline

Drop license-safe assets here.

- `textures/`: optional PNG/JPG/BMP textures
- `models/`: optional Panda3D-ready models (`.bam`, `.egg`, `.glb`, `.gltf`)
- `animations/`: optional animation clips or source files
- `icons/items/`: 64x64 transparent PNG item icons referenced as `items/<name>`
- `icons/skills/`: 64x64 transparent PNG skill icons referenced as `skills/<name>`
- `icons/ui/`: 64x64 transparent PNG UI icons referenced as `ui/<name>`
- `sprites/effects/`: transparent PNG effect sprites referenced as `effects/<name>`

`asset_manifest.json` is the source of truth for logical asset keys and their
relative files. Keep icon keys lowercase, use underscores for multi-word names,
and do not include a file extension in gameplay data. Example item data:

```json
"icon": "items/sword"
```

## Placeholder Icons

Run the generator from the repository root:

```powershell
python -m game.tools.generate_placeholder_icons
```

The compatibility module path also works:

```powershell
python -m hearthvale.game.tools.generate_placeholder_icons
```

The generator reads `asset_manifest.json` and creates deterministic 64x64
transparent PNG placeholders for every item, skill, UI, and effect entry. Audio
entries are resolved by the runtime but are not generated by this PNG tool. The
generator is safe to run repeatedly and skips existing PNGs by default. Pass
`--force` only when you want to replace existing placeholder or final art files.

## Asset Audit

Run the asset audit from the repository root:

```powershell
python -m game.tools.asset_audit --write
```

It reports missing manifest files, item and skill icon coverage, orphaned
manifest-backed art, and direct texture/model/animation inventory so you can
spot content gaps before they become gameplay regressions.

## Replacing Placeholders

Final artwork should be original, license-safe, transparent PNG, and 64x64
pixels unless the manifest is intentionally changed. Replace a placeholder by
dropping the final PNG at the same manifest path. Keep the same logical key in
data files so saves and UI code do not need to change.
