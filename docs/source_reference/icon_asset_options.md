# Icon Asset Options

Use only assets with a clear source page and saved license text in the repo before importing them.

| Source/site | License | Attribution required | Commercial use | Recommended use case |
| --- | --- | --- | --- | --- |
| [Kenney assets](https://kenney.nl/assets) | CC0 public domain for game asset pages per [Kenney support](https://kenney.nl/support) | No | Yes | First choice for UI, inventory, and simple game icons where a Kenney pack fits the style. |
| [OpenGameArt](https://opengameart.org/) assets filtered to CC0 only | Per-asset; choose CC0 only unless attribution support is added. OGA's FAQ says CC0 can be used without asking or crediting. | No for CC0 | Yes for CC0 | RPG item, skill, and world-object icons after verifying each individual asset page. |
| [Lucide](https://lucide.dev/license) | ISC; some icons derived from Feather are MIT | Include license/copyright notice in distributed copies | Yes | Clean UI symbols such as close, settings, inventory tabs, menu, search, and arrows. |
| [Heroicons](https://heroicons.com/) | MIT | Include license/copyright notice in distributed copies | Yes | Alternative general UI SVG icons if the visual style fits better than Lucide. |
| [Game-icons.net](https://game-icons.net/) | CC-BY or public domain for some icons | Yes for CC-BY; credit must be accessible in game credits or docs | Yes, with attribution | Strong fantasy/skill/item symbols only after an attribution screen/file is implemented, or when an individual icon is public domain. |
| [CraftPix](https://craftpix.net/file-licenses/) | Site license for free and paid products; not open source | No attribution required, but source-file redistribution is restricted | Yes under license terms | Paid or free polished icon packs if the project accepts the redistribution limits and keeps license proof. |

Avoid ripped assets, lookalike assets from copyrighted games, unclear AI-generated packs, and assets marked NC, ND, editorial-only, or personal-use-only.
